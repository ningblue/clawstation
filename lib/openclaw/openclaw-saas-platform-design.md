# OpenClaw SaaS 多租户平台设计方案

## 一、核心概念模型

```
┌─────────────────────────────────────────────────────────────────────────────────────────────┐
│                              租户-用户-资源关系模型                                           │
├─────────────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                             │
│   平台 (Platform)                                                                           │
│   └── 租户 A (Tenant: 公司A)                                                                │
│       ├── 用户 A1 (Owner)      ← 租户创建者，拥有所有权限                                    │
│       ├── 用户 A2 (Admin)      ← 管理员，可以管理资源和用户                                  │
│       └── 用户 A3 (Member)     ← 普通成员，只能使用资源                                      │
│       │                                                                                     │
│       └── 资源池 (属于租户A)                                                                 │
│           ├── OpenClaw Instance #1 (生产环境)                                               │
│           └── OpenClaw Instance #2 (测试环境)                                               │
│                                                                                             │
│   └── 租户 B (Tenant: 公司B)                                                                │
│       ├── 用户 B1 (Owner)                                                                   │
│       └── 用户 B2 (Member)                                                                  │
│       │                                                                                     │
│       └── 资源池 (属于租户B)                                                                 │
│           └── OpenClaw Instance #1                                                         │
│                                                                                             │
│   └── 租户 C (Tenant: 个人开发者C)                                                          │
│       └── 用户 C1 (Owner)  ← 个人租户只有创建者一人                                         │
│           │                                                                                 │
│           └── 资源池 (属于租户C)                                                             │
│               └── OpenClaw Instance #1                                                     │
│                                                                                             │
│  ═══════════════════════════════════════════════════════════════════════════════════════   │
│  关键原则:                                                                                  │
│  • 资源 (OpenClaw Instance) 属于租户，租户内所有用户共享                                     │
│  • 用户通过租户来访问资源                                                                    │
│  • 租户之间完全隔离（命名空间、网络、存储）                                                  │
│  • 一个用户只能属于一个租户（简化设计）                                                      │
│                                                                                             │
└─────────────────────────────────────────────────────────────────────────────────────────────┘
```

## 二、数据模型设计

### 2.1 租户模型 (Tenant)

```typescript
interface Tenant {
  // 基础信息
  id: string;                    // 租户唯一标识 (t_xxxxxxxx)
  name: string;                  // 租户名称（公司名称）
  slug: string;                  // URL 标识（如 company-a）
  type: 'personal' | 'team' | 'enterprise';  // 租户类型
  status: 'active' | 'suspended' | 'terminated';
  
  // 订阅套餐
  subscription: {
    planId: string;              // 套餐ID: free/startup/business/enterprise
    planName: string;            // 套餐名称
    status: 'trial' | 'active' | 'expired' | 'cancelled';
    startedAt: Date;
    expiresAt?: Date;
    autoRenew: boolean;
  };
  
  // 资源配额（由套餐决定，可自定义）
  quota: {
    maxInstances: number;        // 最大实例数
    maxUsers: number;            // 最大用户数
    maxChannelsPerInstance: number;  // 每实例最大渠道数
    maxStorageGB: number;        // 总存储配额
    maxMonthlyMessages: number;  // 月消息配额
  };
  
  // 资源配置（实际分配）
  resources: {
    k8sNamespace: string;        // K8s 命名空间
    domain: string;              // 专属域名: company-a.ocr.openclaw.cloud
    nodePool?: string;           // 专属节点池（企业版）
  };
  
  // 品牌定制（企业版）
  branding?: {
    logo?: string;
    primaryColor?: string;
    customDomain?: string;
  };
  
  // 统计
  stats: {
    userCount: number;
    instanceCount: number;
    totalMessages: number;
    storageUsedGB: number;
    createdAt: Date;
    lastActivityAt: Date;
  };
  
  // 计费
  billing: {
    country: string;             // 用于税务
    currency: string;            // CNY/USD
    paymentMethod?: string;
    balance: number;             // 余额（预付费）
  };
}
```

### 2.2 用户模型 (User)

```typescript
interface User {
  // 基础信息
  id: string;                    // 用户唯一标识 (u_xxxxxxxx)
  tenantId: string;              // 所属租户ID
  
  // 登录凭证
  email: string;                 // 登录邮箱（全局唯一）
  phone?: string;                // 手机号
  passwordHash: string;          // 密码哈希
  
  // 个人信息
  profile: {
    name: string;                // 显示名称
    avatar?: string;             // 头像URL
    department?: string;         // 部门
    jobTitle?: string;           // 职位
  };
  
  // 角色权限
  role: 'owner' | 'admin' | 'member' | 'viewer';
  permissions: Permission[];     // 细粒度权限（可覆盖角色默认权限）
  
  // 状态
  status: 'active' | 'inactive' | 'suspended';
  lastLoginAt?: Date;
  lastLoginIp?: string;
  
  // 偏好设置
  preferences: {
    language: string;            // zh-CN / en-US
    timezone: string;
    notifications: NotificationPrefs;
  };
  
  createdAt: Date;
  updatedAt: Date;
}

// 角色权限定义
const ROLE_PERMISSIONS = {
  owner: {
    description: '租户创建者，拥有所有权限',
    permissions: ['*'],  // 所有权限
    canManageUsers: true,
    canDeleteTenant: true,
  },
  admin: {
    description: '管理员，可以管理资源和用户',
    permissions: [
      'instance:create', 'instance:read', 'instance:update', 'instance:delete',
      'channel:create', 'channel:read', 'channel:update', 'channel:delete',
      'user:create', 'user:read', 'user:update', 'user:delete',
      'billing:read',
      'settings:read', 'settings:update',
    ],
    canManageUsers: true,
    canDeleteTenant: false,
  },
  member: {
    description: '普通成员，可以配置渠道和使用功能',
    permissions: [
      'instance:read',
      'channel:create', 'channel:read', 'channel:update', 'channel:delete',
      'conversation:read', 'conversation:send',
      'settings:read',
    ],
    canManageUsers: false,
    canDeleteTenant: false,
  },
  viewer: {
    description: '观察者，只读权限',
    permissions: [
      'instance:read',
      'channel:read',
      'conversation:read',
      'settings:read',
    ],
    canManageUsers: false,
    canDeleteTenant: false,
  },
};
```

### 2.3 资源实例模型 (Instance)

```typescript
interface OpenClawInstance {
  // 基础信息
  id: string;                    // 实例ID (i_xxxxxxxx)
  tenantId: string;              // 所属租户
  name: string;                  // 实例名称（如"生产环境"、"测试环境"）
  description?: string;
  
  // 规格配置
  spec: {
    templateId: string;          // 资源模板ID
    resources: {
      cpu: string;               // 如 "2"
      memory: string;            // 如 "4Gi"
      storage: string;           // 如 "20Gi"
    };
  };
  
  // 部署信息
  deployment: {
    namespace: string;           // K8s 命名空间（继承租户）
    podName: string;             // Pod 名称
    nodeName: string;            // 所在节点
    version: string;             // OpenClaw 版本
    gatewayUrl: string;          // 访问地址
    internalIp: string;          // 集群内IP
    status: 'pending' | 'creating' | 'running' | 'stopping' | 'stopped' | 'error' | 'upgrading';
    health: 'healthy' | 'unhealthy' | 'unknown';
    createdAt: Date;
    startedAt?: Date;
  };
  
  // 功能配置
  config: {
    channels: ChannelConfig[];   // 渠道配置
    models: ModelConfig;         // AI 模型配置
    agents: AgentConfig[];       // Agent 配置
    tools: ToolConfig;           // 工具权限
    security: SecurityConfig;    // 安全策略
  };
  
  // 使用统计
  stats: {
    messageCount: number;        // 总消息数
    channelCount: number;        // 渠道数
    lastActivityAt?: Date;
  };
  
  // 访问控制
  accessControl: {
    allowedUsers: string[];      // 允许访问的用户ID（为空表示全部）
    ipWhitelist?: string[];      // IP 白名单
  };
}
```

## 四、核心业务流程

### 4.1 租户注册流程

```
用户访问 https://app.openclaw.cloud
        │
        ▼
┌─────────────────────────────────────┐
│ 1. 填写注册信息                      │
│    • 邮箱 / 密码                     │
│    • 租户名称（公司名）               │
│    • 选择套餐（Free Trial）          │
└─────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────┐
│ 2. 创建租户                          │
│    • 生成 tenantId: t_xxxxxx         │
│    • 生成 slug: company-name         │
│    • 分配配额（基于套餐）            │
│    • 创建 K8s Namespace              │
└─────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────┐
│ 3. 创建用户（Owner）                 │
│    • 绑定到租户 t_xxxxxx             │
│    • 角色: Owner                     │
│    • 拥有全部权限                    │
└─────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────┐
│ 4. 部署默认实例                      │
│    • 创建 OpenClaw Instance          │
│    • URL: company-name.ocr...        │
│    • 初始配置（空的）                │
└─────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────┐
│ 5. 完成引导                          │
│    • 发送欢迎邮件                    │
│    • 跳转到用户控制台                │
│    • 显示快速入门向导                │
└─────────────────────────────────────┘
```

## 七、多租户隔离机制

### 7.1 数据隔离

| 层级 | 隔离方式 | 说明 |
|------|---------|------|
| **数据库** | Schema-per-tenant | 每个租户独立的 PostgreSQL Schema |
| **缓存** | Key prefix | Redis key 格式: `tenant:{id}:...` |
| **对象存储** | Bucket prefix | S3 path: `/tenants/{id}/...` |
| **日志** | 标签过滤 | Loki label: `tenant_id={id}` |

### 7.2 K8s 隔离

| 层级 | 隔离方式 | 说明 |
|------|---------|------|
| **Namespace** | 每租户一个 | `tenant-t-xxxxxx` |
| **网络** | NetworkPolicy | 禁止跨租户通信 |
| **资源** | ResourceQuota | 限制租户总资源使用 |
| **存储** | PVC + StorageClass | 独立存储卷 |

### 7.3 认证隔离

```typescript
// JWT Token 结构
interface AccessToken {
  // 标准声明
  sub: string;                   // userId
  iat: number;
  exp: number;
  
  // 自定义声明
  tenantId: string;              // 所属租户
  tenantSlug: string;            // 租户标识
  role: string;                  // 用户角色
  permissions: string[];         // 权限列表
  instanceAccess: string[];      // 可访问的实例（为空表示全部）
}

// API 权限校验中间件
async function authorize(req: Request, requiredPermission: string) {
  const token = decodeJWT(req.headers.authorization);
  
  // 1. 检查用户是否有指定权限
  if (!token.permissions.includes(requiredPermission) && 
      !token.permissions.includes('*')) {
    throw new ForbiddenError('Permission denied');
  }
  
  // 2. 检查租户状态
  const tenant = await getTenant(token.tenantId);
  if (tenant.status !== 'active') {
    throw new ForbiddenError('Tenant is not active');
  }
  
  // 3. 设置请求上下文
  req.tenantId = token.tenantId;
  req.userId = token.sub;
  req.userRole = token.role;
}
```

## 八、总结

这个设计方案建立了正确的 **多租户 SaaS 模型**：

### 核心关系
```
Platform (1)
  └── Tenant N (公司/组织)
        ├── User 1 (Owner)
        ├── User 2 (Admin)  
        ├── User 3 (Member)
        │
        └── Resource (OpenClaw Instance) ← 租户内所有用户共享
```

### 关键设计
| 方面 | 设计 |
|------|------|
| **租户** | 资源隔离边界，有自己的套餐、配额、命名空间 |
| **用户** | 属于租户，有角色（Owner/Admin/Member/Viewer）|
| **资源** | 属于租户，租户内用户根据权限访问 |
| **隔离** | K8s Namespace + NetworkPolicy + ResourceQuota |
| **数据** | PostgreSQL Schema-per-tenant |

### 使用场景
1. **个人用户** - 创建个人租户，自己当 Owner
2. **小团队** - 创建团队租户，邀请 3-5 个成员
3. **企业** - 创建企业租户，邀请 50+ 员工，多个实例（生产/测试）

这个平台设计支持从个人到大型企业的多租户场景，资源属于租户，用户在租户内协作共享资源。
